# Guides

Guides are **planned playbooks** (day-0/day-1/day-2) that walk through an end-to-end workflow, often spanning multiple components.

Rules of thumb:
- If the doc is “what to do when an alert fires / during an incident”, it belongs in `docs/runbooks/`.
- If the doc is a narrow operational recipe or troubleshooting note that isn’t tied to a specific alert, it belongs in `docs/toils/`.

Useful entry points:
- `docs/guides/bootstrap-new-cluster.md` for the end-to-end cluster bootstrap playbook
- `docs/guides/public-mirror-preparation.md` for preparing a sanitized public mirror
