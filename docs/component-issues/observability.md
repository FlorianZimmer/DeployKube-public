- Grafana OIDC login redirected to Keycloak but Keycloak showed “Client not found” because the `deploykube-admin` realm did not define a `grafana` OIDC client. Fix: add the Grafana client to the realm template and project the client secret into the `keycloak` namespace for bootstrap rendering.
- Grafana OIDC login failed with “Sign up is disabled” because `GF_AUTH_GENERIC_OAUTH_ALLOW_SIGN_UP` was disabled, so Grafana refused to auto-provision the user on first login. Fix: enable OIDC signup while keeping the local login form disabled.
- Grafana `RollingUpdate` was blocked by `CrashLoopBackOff` with `Datasource provisioning error: data source not found` due to datasource UID drift vs. an existing Grafana DB. Fix: delete and recreate the GitOps-managed datasources on startup via `values.yaml` (`deleteDatasources:`) to keep names/Uids consistent across revisions.

## Evidence (2025-12-15)
- Grafana **Cluster Overview** dashboard showed no data because the metrics pipeline was effectively empty:
  - Alloy chart values used an outdated schema (`configMap.*` instead of `alloy.configMap.*`), so the DaemonSet deployed with the chart’s default config (no scrapes/remote_write).
  - `node-exporter` and `kube-state-metrics` were missing, so `node_*` and `kube_*` metrics were absent.
  - `node-exporter` defaulted to `hostNetwork: true`, which broke scrapes under deny-by-default NetworkPolicies (targets resolved to host IPs).
  - Observability namespace could not reach kube-apiserver (service endpoints are node IPs); fixed with a `CiliumNetworkPolicy` allow rule.
- Fix: added `components/platform/observability/metrics`, corrected Alloy chart values/config, configured `node-exporter` to run on the pod network, and allowed kube-apiserver egress in `observability` via `CiliumNetworkPolicy`.
- Follow-up: standard Kubernetes dashboards also require kubelet/cAdvisor (`container_*`, `machine_*`) and apiserver (`apiserver_*`) metrics. Implemented a dedicated `alloy-metrics` Deployment (single writer) to scrape these via the apiserver proxy (RBAC `nodes/proxy` + `/metrics`) and remote_write into Mimir, avoiding `out-of-order samples` caused by DaemonSet replicas scraping the same targets.
