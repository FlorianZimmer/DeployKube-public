## Resolved Issues

### 2026-03-01 — Proxmox + mac-orbstack-single authZ hardening for Hubble oauth2-proxy

- Added fail-closed group-gating in:
  - `networking/hubble/overlays/proxmox-talos/oauth2-proxy-deployment.yaml`
  - `networking/hubble/overlays/mac-orbstack-single/oauth2-proxy-deployment.yaml`
  - `--oidc-groups-claim=groups`
  - `--allowed-group=dk-platform-admins`
  - `--allowed-group=dk-platform-operators`
  - `--allowed-group=dk-security-ops`
- Removed secret-bearing oauth2-proxy args (`--client-secret=...`, `--cookie-secret=...`) and rely on native env var configuration.
- Added `groups` mapper to the Keycloak Hubble client in the realm template to ensure allowlist claims are emitted.
-

### 2026-01-07 — `hubble-ui` Service is ClusterIP (no NodePort)

- Updated Stage 0 Cilium defaults to avoid NodePort exposure and added a GitOps PostSync hook (`Job/hubble-ui-service-posture`) to enforce `kube-system/svc/hubble-ui` is `ClusterIP`.

### 2026-01-07 — Ingress reachability smoke exists (global)

- `CronJob/ingress-smoke-substrate` validates that all HTTPRoutes attached to `Gateway/public-gateway` resolve and are reachable via the ingress VIP (includes `hubble.*`).

### 2026-01-08 — OIDC auth proxy enabled

- Added `oauth2-proxy` in front of Hubble UI and switched the `hubble-ui` HTTPRoute to route to the proxy (Keycloak OIDC).
