- **2026-01-07 – Kiali OIDC enabled:** switched Kiali from anonymous access to Keycloak OIDC (`auth.strategy: openid`) and added a PostSync smoke to assert the auth config and client secret projection.
- **2025-12-29 – Istio operator blocked by access-guardrails + RBAC lock:** `IstioOperator/istio-control-plane` stayed in `ERROR` because the in-cluster `istio-operator` reconciler uses server-side apply on Istio CRDs/webhooks and required RBAC objects, which was denied by `deploykube-access-guardrails-self-protection` and `deploykube-rbac-mutation-lock`. Fix: add narrow allowlist exceptions for the `istio-operator` service account.
- **2025-12-29 – Sidecar injector webhook broken (apiserver x509 unknown authority):** the apiserver could not call `namespace.sidecar-injector.istio.io` because `istiod` was blocked from patching the `caBundle` into `MutatingWebhookConfiguration/istio-sidecar-injector` and `ValidatingWebhookConfiguration/istio-validator-istio-system`. Fix: allowlist `system:serviceaccount:istio-system:istiod` for `UPDATE` on `istio-*` webhook configurations in the self-protection guardrail.
- **2025-12-29 – Dev MetalLB IP mismatch for the ingress gateway:** dev was referencing the prod load balancer IP (`198.51.100.63`) in the `IstioOperator` spec, which is outside the OrbStack MetalLB pool. Fix: remove the fixed IP from the base `IstioOperator` spec so dev can auto-assign from `orbstack-pool`, and re-add the prod-only pinned IP via the `networking-istio-control-plane` app’s `spec.source.kustomize.components` (`env-components/prod`).
- **Argo CD deadlock under mesh STRICT (Cilium kube-proxy replacement + TCP):** Argo CD’s `repo-server` (gRPC/TCP) and `redis` (TCP) traffic can fall back to Envoy `PassthroughCluster` (plaintext) and gets rejected by mesh-default `PeerAuthentication` STRICT, breaking manifest generation (`tls: certificate required`). Fix: add selector-scoped `PeerAuthentication` PERMISSIVE for `argocd` `argocd-repo-server` and `argocd-redis` via `networking/istio/mesh-security` (Dec 25, 2025; evidence: ).
- **Admission webhooks timing out under mesh STRICT:** The Kubernetes apiserver is not in-mesh, so sidecar-injected admission webhook services can time out during validation. Fix: add selector-scoped `PeerAuthentication` PERMISSIVE for `cert-manager` and `external-secrets` webhook workloads via `networking/istio/mesh-security` (Dec 25, 2025; evidence: ).
- **ExternalDNS to PowerDNS API blocked by STRICT mTLS:** ExternalDNS is mesh-injected but PowerDNS runs without a sidecar (UDP/53), so Envoy attempted TLS to the plaintext API and returned `WRONG_VERSION_NUMBER`, causing ExternalDNS CrashLoopBackOff. Fix: add a scoped DestinationRule disabling mTLS for `powerdns.dns-system.svc.cluster.local:8081` (Dec 20, 2025).
- **In-cluster Kubernetes API access blocked by Istio mTLS:** Jobs and scripts that call the kube-apiserver (`kubectl` in hooks/cronjobs) were failing with TLS errors. Added a mesh-level DestinationRule in `istio-system` to disable mTLS for `kubernetes.default.svc.cluster.local` so the API server stays reachable from injected workloads (Dec 19, 2025).
- **Mesh security verifier blocked by failed Job pods:** the PreSync verifier treated failed Job/CronJob pods as readiness failures, preventing sync when smoke tests or hooks failed. Updated the verifier script to ignore Job-owned pods and only validate long-running workloads (Dec 19, 2025).
- **Gateway ingress + STRICT mTLS**: External browsers lost access to Forgejo/Argo/Keycloak once mesh-wide STRICT rolled out because the ingress gateway inherited the same policy. Added `PeerAuthentication ingressgateway-permissive` (namespace `istio-system`) so north-south TLS stays plaintext-to-Gateway while east-west traffic remains STRICT (Nov 16, 2025).
- **Namespace labeling drift**: all platform namespaces are now declared in `networking/istio/namespaces` and applied at sync wave `-2`, so Stage 1 no longer races `CreateNamespace=true` against injection labels (Nov 15, 2025).
- **Mesh enforcement guardrails**: `networking/istio/mesh-security` adds the STRICT `PeerAuthentication`, mesh-default `DestinationRule`, plaintext-deny `AuthorizationPolicy`, and the PreSync `mesh-security-verifier` Job that blocks Argo syncs until every required namespace has ready sidecars (Nov 15, 2025).
