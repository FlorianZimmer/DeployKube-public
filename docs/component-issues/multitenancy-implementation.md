## Scope (Queue #11: Tier S MVP)

This implementation tracker is scoped to **Tier S (shared-cluster multitenancy)** only: logical isolation enforced by **labels + admission + policy + Argo boundaries + RBAC**.

Non-goal (for Queue #11): implementing Tier D/H (dedicated clusters and/or hardware separation).

Compatibility requirement (do not block Tier D/H later):
- Keep `orgId` / `projectId` stable and tier-agnostic (no `deploymentId`/cluster/backend encoding).
- Keep Git contracts portable: “move tenant/project to dedicated” should be achievable by changing platform-owned registry/destination wiring, not rewriting tenant identity.
- Keep guardrails layered and reusable (VAP/Kyverno/Argo/RBAC) so dedicated clusters can reuse the same enforcement model.

## Promotion gates (Tier S0 → S1)

These are the “do not claim hostile tenants” gates; treat them as a hard checklist.

- [x] Identity labels enforced + immutable (M1).
- [x] Argo default project deny-by-default; tenant AppProjects constrain destinations and prohibited kinds (M2).
- [x] Tenant PR gate suite is a required PR check in tenant repos (CI + Forgejo protected branches) (M2).
- [x] Tenant RBAC propagation does not permit namespace creation/label mutation and cannot grant ESO/admission/RBAC verbs (M3).
- [x] Ingress hijack prevention + tenant NetPol guardrails + mesh posture validated (M4).
- [x] Tenant ESO usage denied (M5 Phase 1).
- [x] Storage backend reachability constrained (Garage NetPol + PVC constraints) (M6 baseline).
