### Critical
#### general-security-pass
- `rbac-namespace-sync` can bind arbitrary `ClusterRole`s (cluster-admin escalation) and reconcile RBAC across namespaces; restrict `bind` via `resourceNames` to only intended template roles, drop unused RBAC writes (e.g. `roles`), and consider an admission guard that only allows managed bindings. (ids: `dk.ca.finding.v1:shared-rbac:90ac8f753563d05bbc30410ec21ab2764540f02931069e2001ba9da201b24722`, `dk.ca.finding.v1:shared-rbac:0671a371e8bef01d3da1b24f95848ce8d229a427c63a606bd0603ceb1fcd63a3`, `dk.ca.finding.v1:shared-rbac:c43dbfcc525e204685def7915dddd1c829cff1e4f9a3adb59677ffe2ee96b1dc`, `dk.ca.finding.v1:shared-rbac:0c646f8d525d7f4b58ce1a426884edaa2d83636b08461c7e2c901509886159e0`)

### High
#### general-security-pass
- `rbac-system` runs with `pod-security.kubernetes.io/enforce=privileged` and Jobs/CronJobs lack hardening/non-root `securityContext`; tighten PSA to baseline/restricted (with minimal documented exceptions) and set `runAsNonRoot`, non-zero `runAsUser`, `allowPrivilegeEscalation: false`, `capabilities.drop: ['ALL']`, `seccompProfile: RuntimeDefault`. (ids: `dk.ca.finding.v1:shared-rbac:60f9a3f5f23709427d8e2d450bd641dc4fcce94ad142df95ab9a44dd031c1724`, `dk.ca.finding.v1:shared-rbac:8cc2f8fa60d62584849652bdc13fbd592908e20632a5b604648297053ab2c54d`, `dk.ca.finding.v1:shared-rbac:7b7390d28e6a17dd6243befbb570305c62b1a7e0850979a21c4f77f3c2978fed`, `dk.ca.finding.v1:shared-rbac:6f96a23460711428a03b67c943f5a032811a6bb47999282ca09c71550240f0f1`, `dk.ca.finding.v1:shared-rbac:581abcc36eb6f7d1f25ad854bf90935b8d5e777665d23e72bdf0023f4ef2e4ed`)

#### rbac-implementation
- `auditor-readonly` effectively grants namespace-wide read of all resources including `Secrets`; split into a safer default auditor role (no secrets) and only grant secret-read explicitly where needed. (ids: `dk.ca.finding.v1:shared-rbac:2b54ef4713709f9497f6a762556cf20b27170db7d04d8cfb6eaeee3ab19e1dbf`)
- Standing `cluster-admin` binding for platform admins is permanent in manifests; keep breakglass offline-only and consider separating day-to-day admin roles from `cluster-admin`. (ids: `dk.ca.finding.v1:shared-rbac:510d78def9a8eb90b6dad1d0226aa9afe41c3f76c9540ff0004a4481dfb492f8`)

### Medium
#### auth-flow
- PR gate enforcement can silently skip when credentials/inputs are missing; fail closed (non-zero) and/or always emit an explicit status signal (e.g. ConfigMap) so skipped enforcement is visible and alertable. (ids: `dk.ca.finding.v1:shared-rbac:1dccb87d95a747f6743f64ee1da9de571aff2b8ddb47f84df03b26d15e12e65f`)

#### capacity-and-performance-hygiene
- Scheduled Jobs/CronJobs (and the smoke Job) have no `resources.requests/limits`; add conservative requests and bounded limits to reduce retry-storm/noisy-neighbor risk and improve scheduling predictability. (ids: `dk.ca.finding.v1:shared-rbac:8ba5c79a3f043cde8b7fea55dfb15e335ff95197bc2cd8712c70635de4c18cbc`, `dk.ca.finding.v1:shared-rbac:4b26ae1c8699ebbd51cb433c2ec007dc1f63e9b6aae00cf34671aa6bf7e65691`, `dk.ca.finding.v1:shared-rbac:59aa081354033e5e75c7368456b13586074bbe02713e45431c0a0d55ed714be7`)

#### design-doc-vs-implementation-drift
- Design doc says `forgejo-team-sync` runs in `forgejo` namespace but manifests run it in `rbac-system`; update `docs/design/rbac-architecture.md` to match runtime placement (and rationale if needed). (ids: `dk.ca.finding.v1:shared-rbac:1ade6bfc4debce457aec90e2dae80d972c0dd8585b67c9539d79c08c1d524c61`)

#### documentation-coverage-and-freshness
- `platform/gitops/components/shared/rbac/README.md` still needs a short component-specific upgrade/rollback note beyond version alignment. (ids: `dk.ca.finding.v1:shared-rbac:8640f6a052a09c9b3e727ae941133cfc811a42f3c60bc8980ddbfb9713cd83d3`)

#### general-security-pass
- Images are tag-referenced without immutable digests; pin images by digest and align `imagePullPolicy` with that strategy (digest + `IfNotPresent`, or tag + `Always`). (ids: `dk.ca.finding.v1:shared-rbac:4a789a1629df3b43454f9ba278de19e0845ebc89ebb71a10f361c6dcc4b49296`, `dk.ca.finding.v1:shared-rbac:893343a005da61d61ef8bec120a90cf05405d1819d4245e6caf61b713036c7f8`, `dk.ca.finding.v1:shared-rbac:b89edccb5a2df47b56b5ed0de22e99e7d77a054327ea2f3024a247376b81bc43`, `dk.ca.finding.v1:shared-rbac:f9eba44684cb29dddaa52cd6a2ae67dbd1f4abf986b0c0d11adf132bc14f081f`)

#### high-availability
- CronJobs lack explicit overlap/catch-up controls (`concurrencyPolicy`, `startingDeadlineSeconds`) and `rbac-namespace-sync` lacks a runtime deadline; set these knobs (and optionally a bounded `backoffLimit`) to avoid overlapping runs and unbounded catch-up after outages. (ids: `dk.ca.finding.v1:shared-rbac:0b682b559131b87d860838c02413929b6fa1b6227796fb03f5d5ea705f16d53f`, `dk.ca.finding.v1:shared-rbac:206d2b45593bb2c39191fceff8458b0b48981dca98b823bcdd58f5302dc18d72`, `dk.ca.finding.v1:shared-rbac:19c4639517d478238e031e99ab89e741cbb790152a18b2d202c8c7efa318664c`)

#### secrets-handling
- Multiple credentials are injected via environment variables; prefer projected secret volumes + file reads at runtime and disable unused auth paths (token vs basic) where possible. (ids: `dk.ca.finding.v1:shared-rbac:c8ee09b5850d6737dfb98dbbcacd722aea033beb823537942638f10224cef11b`)

### Low
#### high-availability
- Forgejo CronJobs set `backoffLimit=0`; consider a small non-zero backoff (e.g. 1–2) to reduce missed-sync risk during transient failures. (ids: `dk.ca.finding.v1:shared-rbac:0c690cb611eb74d3c39896408f623858ea27bf21ef0f9c2f329010f8f8dbc773`)

- **2026-03-01 – Removed OIDC breakglass cluster-admin binding from shared RBAC:** deleted `ClusterRoleBinding/breakglass-cluster-admin` from GitOps manifests, updated smoke expectations, and validated prune convergence on Proxmox and OrbStack.
- **2026-03-01 – Closed temporary group mapping drift:** `forgejo-team-sync` now consumes canonical `dk-*` personas (`dk-platform-admins`, `dk-platform-operators`, `dk-auditors`) rather than `temp-*` groups.

- **In-cluster Forgejo HTTPS cut-over for RBAC sync jobs (2026-02-28):** moved `forgejo-team-sync` and `forgejo-tenant-pr-gate-enforcer` to `https://forgejo-https.forgejo.svc.cluster.local`, added explicit Forgejo CA validation (`forgejo-repo-tls`), and updated `rbac-system` egress policy + scoped TLS-secret read RBAC for the sync service account.

- **Tenant PR gate enforcement (2026-01-21):** added `rbac-system/CronJob/forgejo-tenant-pr-gate-enforcer` to converge tenant repo protected-branch required status checks (`tenant-pr-gates`) from the tenant registry (requires `Secret/rbac-system/forgejo-admin`).

- **Tier S hardening evidence pack (2026-01-21):** captured repo-level proof that RBAC smokes ship and `rbac-system` has default-deny NetworkPolicies (plus egress allowlist).

- **Namespace RBAC sync scale-safety (2026-01-20):** `rbac-namespace-sync` now annotates managed RoleBindings with a hash and skips unchanged resources, with retry/backoff under transient API pressure. `rbac-smoke` also asserts tenant workload ClusterRoles do not grant access-plane APIs and that tenant bindings are hash-annotated.

- **Forgejo team sync degraded (2026-01-20):** root cause was the `forgejo-team-bootstrap` job being unable to write the team-sync PAT to Vault (missing `rbac-system` → `vault-system` egress) and reusing tokens that were valid for `/api/v1/user` but insufficient for org/team operations, resulting in 401s. Fix: allow egress to Vault and tighten token validation to rotate on insufficient permissions.

- **Tenant RBAC namespace sync (2026-01-14):** `rbac-namespace-sync` now supports `darksite.cloud/rbac-profile=tenant` (`tenant-id` + `project-id` → RoleBindings), and the `rbac-smoke` hook asserts tenant template rendering.

- **RBAC-system egress allowlist (2026-01-10):** Added default-deny NetworkPolicies to `rbac-system` with an explicit egress allowlist (DNS, Istio control-plane, Keycloak, Forgejo) and a CiliumNetworkPolicy for kube-apiserver egress.

- **RBAC smoke gate (sync, 2026-01-07):** Added `Job/rbac-smoke` PostSync hook under `platform/gitops/components/shared/rbac/tests/` to assert required ClusterRoles/Bindings and CronJob presence.
- **Forgejo team sync TLS + reachability (2025-12-19):** `forgejo-team-sync` was using in-cluster service DNS names that did not match Step CA certificate SANs, and external LB hairpin was unreachable from inside the cluster. Updated the CronJob to use the in-cluster Forgejo HTTP service (mTLS via Istio) and to keep Keycloak on the external hostname while resolving it to the in-cluster service (`KEYCLOAK_CONNECT_HOST`) so TLS/SNI stays valid without LB hairpin.
- **Forgejo team sync early-exit robustness (2025-12-29):** when credentials were missing (or Istio injection was temporarily broken), the CronJob’s early-exit path could still end up with a failed pod. Fix: make the Istio sidecar quit trap best-effort (`deploykube_istio_quit_sidecar || true`) so “skip” paths can still exit `0`.
- **Forgejo team sync stdout contamination (2025-12-29):** `ensure_team` logs were written to stdout and polluted command substitutions (team IDs), causing `curl: (3) bad range in URL` and spurious job retries. Fix: write `forgejo-team-sync` log output to stderr and limit job retries/history to reduce failed-pod churn.
- **Forgejo team bootstrap PAT reliability (2025-12-19):** avoid deleting the active PAT before a successful end-to-end reconcile. The bootstrap job now reuses the existing token by default; when a new token is minted it uses a timestamped name and prunes old tokens only after Vault/Kubernetes have been updated. (Now owned by `shared-rbac-secrets`.)
- **Forgejo team bootstrap Keycloak TLS trust (2025-12-19):** the bootstrap job now pulls the Keycloak CA bundle from the in-cluster `forgejo-team-sync` secret (and preserves it when writing back the secret), preventing `curl` TLS failures against the Keycloak HTTPS endpoint. (Now owned by `shared-rbac-secrets`.)
- **Forgejo team sync Keycloak secret drift (2025-12-19):** root cause: the bootstrap job rotated the Keycloak client secret and then hit a transient connection reset, leaving Keycloak updated but Vault/Kubernetes stale. Fix: bootstrap now treats Keycloak as the source of truth and **reconciles** the current client secret back into Vault/Kubernetes; rotation is explicit (`FORGEJO_TEAM_BOOTSTRAP_ROTATE_KEYCLOAK_SECRET=true`). It also verifies stored Keycloak credentials before skipping, so it self-heals if drift is detected. (Now owned by `shared-rbac-secrets`.)
