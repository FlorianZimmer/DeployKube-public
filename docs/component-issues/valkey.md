### High

#### capacity-and-performance-hygiene
- Add explicit `maxmemory` + `maxmemory-policy` to generated config (ideally configurable / derived from container memory limit) to avoid OOM/restart storms. (ids: `dk.ca.finding.v1:valkey:25ea74bc589239e413e94d3ba532afdee969ce9148938f825504e57c5a8ef70d`)

#### high-availability
- Fix client routing semantics: the `valkey` Service selects all pods, so failover/rollouts can send clients to replicas unless every client is Sentinel-aware; document/enforce the contract or provide a safe in-cluster routing pattern to the current master. (ids: `dk.ca.finding.v1:valkey:10a6084abe3ddfc7968773d214fca12237546965c5d8b3fa3568e53521e6e099`)

#### kubernetes-best-practices
- Remove/avoid Argo CD `Replace=true` on the StatefulSet unless a proven immutability edge-case requires it (risk of delete/recreate disruption). (ids: `dk.ca.finding.v1:valkey:c7c17335735ef5b2218495971deec04225650e5ebb3c3c89f440701d951cad0b`)

#### multitenancy-posture
- Move off a single shared password with no authz/per-consumer identity: support per-consumer Secrets (and Valkey ACL users if compatible), and document rotation/revocation expectations. (ids: `dk.ca.finding.v1:valkey:921e5ce0f4bffe65bab26f382c9be69c7d53dd9309052096d5e2cecdb44edc3b`, `dk.ca.finding.v1:valkey:435b40da5bf64543a5e046a3ad681f1a3c9ed54f7d4f3783ae443193082d1a99`)
- Tighten cross-namespace access controls: current namespace-label-only allowlisting is coarse; combine `namespaceSelector` with a `podSelector` (client label) to reduce blast radius and accidental exposure. (ids: `dk.ca.finding.v1:valkey:c51296abcc1cf828657654d50593e5c0584c90dedc8e08b1d4c14021fb49fb10`, `dk.ca.finding.v1:valkey:82df11ac077ec4061c4aec0157f6418847fdeec40c77b0b90f712f88649efa3c`)

#### secrets-handling
- Stop persisting secrets into PVC-backed paths: password is injected via env var and written into generated config under `/valkey` (PVC-backed by default); prefer Secret volume + generate config into `emptyDir`. (ids: `dk.ca.finding.v1:valkey:d05c3093759391e45faeb209f6880a4bc116e3d5b9d3a71c2bf1366203618232`, `dk.ca.finding.v1:valkey:cb4c99205bfcf49f5d8e620f3b73ccb3d5f40a9c1c6e1d70608187f03e89d1d6`)

### Medium

#### backup-and-recovery

- Clarify/enforce transport security: no native TLS is shown; either add an optional TLS mode or explicitly require/enforce mTLS via platform policy and document that dependency. (ids: `dk.ca.finding.v1:valkey:5862cc8bd9225e4744d50ac1f095a44434462e9cad4c2c4f6c94f6591e3f26b3`)

#### kubernetes-best-practices
