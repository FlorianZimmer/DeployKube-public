- **Keycloak OIDC trust CA drift (Dec 19, 2025):** root cause: the CA bundle published via Vault (`secret/keycloak/oidc-ca` → `forgejo-oidc-ca` / `argocd-oidc-ca` / `rbac-system/forgejo-team-sync:ca.crt`) did **not** match the CA that actually signed the Keycloak ingress certificate in this cluster (`cert-manager/step-ca-root-ca`, `CN=Step-Root`). Result: in-cluster Jobs and scripts failed with `x509: certificate signed by unknown authority` / `curl: (60) SSL certificate problem`.
  - Fix: `keycloak-oidc-ca-sync` (in `components/certificates/step-ca/bootstrap`) now mirrors `cert-manager/step-ca-root-ca` into Vault `secret/keycloak/oidc-ca` (Kubernetes auth). ESO fans that CA into the consuming namespaces.

- **`forgejo-oidc-config` Job hanging (Istio sidecar never exits)**: the Forgejo jobs kustomization did not include `components/shared/bootstrap-scripts/istio-native-exit`, so the `istio-native-exit-script` ConfigMap was missing in the `forgejo` namespace. The Job finished its main container, but the Istio sidecar stayed alive and the hook never completed. Fix: add the shared helper ConfigMap to both the base and proxmox overlays and extend the helper to fall back to `wget` when `curl` is unavailable (Dec 18, 2025).
- **`forgejo-oidc-config` Job failing with DNS timeouts**: CoreDNS could not resolve `keycloak.prod.internal.example.com`, so `gitea admin auth add-oauth` failed to reach the OIDC discovery endpoint. Fix: CoreDNS now forwards the prod zone to the PowerDNS ClusterIP and PowerDNS explicitly allows DNS from CoreDNS pods, restoring in-cluster resolution (Dec 19, 2025).
- **`forgejo-team-sync` CronJob flapping on TLS + startup**: the job called Keycloak over HTTPS with the wrong CA bundle and could start before the Istio sidecar was ready, causing `connect: refused` and `x509: unknown authority` failures. Fix: `keycloak-oidc-ca-sync` keeps the Vault CA bundle aligned with cert-manager, the job mounts `keycloak/oidc-ca` and sets `SSL_CERT_FILE`/`CURL_CA_BUNDLE`, and the ExternalSecret refresh interval is short enough to pick up CA updates before the next hourly run (Dec 19, 2025).
- **Argo drift: CNPG admission rejected low-mem Postgres patch**: `platform-forgejo-postgres` drifted `OutOfSync` because the `dev` overlay requested `512Mi` memory while `shared_buffers` was still `1GB`, triggering the CNPG admission check (`requests.memory` must be >= `shared_buffers`). Fix: set low-mem `shared_buffers=256MB` and `max_connections=100` (Dec 15, 2025; evidence: ).
- **Postgres backup CronJob hanging on Istio sidecar**: `postgres-backup` now ensures `curl` exists and calls `istio-native-exit.sh` on exit (with native-sidecar injection) so backup runs reach `Complete` instead of wedging on `istio-proxy` (Dec 14, 2025).
- **Postgres backup CronJob flapping `Connection refused`**: the Job was starting before Envoy bound the redirected ports, so the logical dump command (`pg_dumpall` at the time; now database-only `pg_dump`) hit `Connection refused` and left empty `*-dump.sql` files behind. Fix: set `proxy.istio.io/config: {"holdApplicationUntilProxyStarts": true}` on the pod template and write dumps to a temp file before atomically `mv`-ing into place (Dec 16, 2025; evidence: ).
- **`forgejo-team-sync` CronJob hanging on Istio sidecar**: the CronJob now mounts `istio-native-exit.sh` and traps `deploykube_istio_quit_sidecar` so runs reach `Complete` instead of wedging on `istio-proxy` (Dec 15, 2025; evidence: ).
- **`forgejo-team-bootstrap` failing with Keycloak 401:** the Job requested an admin token using form-encoded `curl -d password=...`, so Vault-managed passwords containing `+` were interpreted as spaces and Keycloak returned `invalid_grant`. Fix: switch to `--data-urlencode` for Keycloak token exchange and remove `set -x` so credentials aren’t echoed into logs (Dec 14, 2025; evidence: ).
- **CrashLoopBackOff during HTTPS cutover:** the Forgejo Deployment was wedged mid-`RollingUpdate` (old pod still Running, new pod CrashLoop) because LevelDB queues on the shared data PVC cannot be opened concurrently (`unable to lock level db at /data/queues/common`). Fix: default the Helm bootstrap install to `strategy.type=Recreate` and make the `forgejo-https-switch` Job temporarily force `Recreate` during its restart (Dec 14, 2025; evidence: `kubectl -n forgejo logs <pod> -c forgejo --previous` contained the lock error; `kubectl -n forgejo get pods` returned a single Forgejo pod `2/2 Running` after switching to `Recreate`).
- **OIDC trust**: the Keycloak issuer CA (`secret/keycloak/oidc-ca`) is now derived from `cert-manager/step-ca-root-ca` (the actual cert-manager signing CA for Keycloak ingress) and fanned out via ESO into `forgejo-oidc-ca` / `argocd-oidc-ca`. This prevents “CA drift” between bootstrap material and the CA that actually issued ingress certificates (Dec 19, 2025).
