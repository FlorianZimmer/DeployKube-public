### High

#### auth-flow
- Replace static PowerDNS `X-API-Key` auth with stronger workload identity where feasible; at minimum scope/rotate the key and ensure it never leaks to logs or process arguments. (ids: `dk.ca.finding.v1:external-sync:3bc6e5e40e451f366ca28cbcb12074d31592739ea66a10cf3671e157b811f3bd`)

#### general-security-pass
- PowerDNS API is accessed over plaintext `http://` (no TLS/mTLS), enabling in-cluster MITM/replay and potential credential exposure; migrate to `https://` with certificate validation (no insecure skips), ideally mTLS. (ids: `dk.ca.finding.v1:external-sync:50596e5e9c97ab1ff19635f07a3fd77cca11fd795f26c399ba2474cabf6ba13b`, `dk.ca.finding.v1:external-sync:48dec6cfc19f2af5006f6ee1806fdc605aac6e551e3ecb35e8d855be394a9692`, `dk.ca.finding.v1:external-sync:c6bb5d3afa600ef9c1a3d344130a97900c0b3116235f44995ef5c4884fb21971`)

#### multitenancy-posture
- `external-sync` is a shared, cluster-scoped actor that can influence tenant DNS (wildcards), so compromise has cross-tenant blast radius; minimize privileges and consider separating or allowlisting tenant DNS publishing. (ids: `dk.ca.finding.v1:external-sync:e94ea1d04e83b11d41745e67252dbd7b3af66d8168b35d50dd1eb545c5f1bfa9`)

#### non-root-containers
- Add explicit pod+container `securityContext` hardening for all Jobs/CronJob (non-root, no priv-esc, drop caps, `seccompProfile: RuntimeDefault`, read-only FS where feasible). (ids: `dk.ca.finding.v1:external-sync:16473c7f8995b674c4f64b3227a5bb83904fa899c260e8a28b62e9aa5491a162`, `dk.ca.finding.v1:external-sync:17e9fcaa08b09f117653e95611f5bf634a18468e1aa3c1b81498c27c0d18df6b`)

### Medium

(ids: `dk.ca.finding.v1:external-sync:144c0a167d1f5c89b3446b94bf20cf3dd7a1d7c9474d6a3fced26985091d72b5`, `dk.ca.finding.v1:external-sync:eaa2fa8be230a63ba5cc2a859e868e53fabb124b793563c134bdbe653c10f997`)

## Resolved Issues

- _2026-03-15_ — **Authoritative drift no longer masked by `dns-sync-status`:** `dns-sync.sh` now treats `dns-sync-status` as an optimization only and verifies the expected authoritative `A`/`NS`/wildcard records still exist in PowerDNS before it skips a republish. This fixes the Proxmox case where `*.prod.internal.example.com` had drifted out of PowerDNS while the status ConfigMap still claimed convergence, which in turn broke `networking-coredns` PostSync wildcard validation until `networking-dns-external-sync` was rerun.
- _2026-03-09_ — **Tracker placeholder removed:** replaced the legacy “None tracked currently.” placeholder with the real tenant wildcard-DNS retirement backlog item so the tracker reflects actual open work.
- _2026-02-22_ — **Authority records decoupled from ingress VIP:** `dns-sync` now uses explicit authority inputs (`DNS_AUTH_NS_HOSTS`, `DNS_AUTH_NS_IP`) from `ConfigMap/deploykube-dns-wiring` for `SOA/NS` and nameserver `A` records, while platform/service `A` records continue to use ingress VIPs. This unblocks correct manual delegation workflows.
- _2026-01-07_ — **Mesh injection posture made explicit:** Removed conflicting annotations from `dns-external-sync` (no injection; no nativeSidecar).
- _2026-01-07_ — **DNS verification added:** Added `PostSync Job/dns-verify` to assert the synced A/wildcard records resolve via CoreDNS and match the current ingress IP (PowerDNS ingress is intentionally restricted to CoreDNS).
- _2026-01-29_ — **DNS wiring controller cutover:** Retired DeploymentConfig-rendered overlays and wired external-sync inputs via tenant provisioner reconciliation. Evidence: `docs/evidence/2026-01-29-dns-wiring-controller-cutover.md`.
