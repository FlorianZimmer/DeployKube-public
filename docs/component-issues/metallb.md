## Resolved Issues

- **2026-02-11 – reserved PowerDNS VIP pool was auto-assigned to non-DNS services (mac-orbstack-single bootstrap deadlock):** `public-gateway-istio` could consume `powerdns-pool` when `autoAssign` was left enabled, causing `dns-system/powerdns` to stall (`AllocationFailed: address also in use`) and bootstrap convergence to loop. Fixes: set `powerdns-pool.spec.autoAssign=false` across overlays, align `mac-orbstack-single` PowerDNS pool to `203.0.113.245`, and reserve legacy/current PowerDNS VIPs in Stage 0 baseline pool to avoid overlap on reruns.
- **2026-01-08 – drift guard fix (RBAC + script):** `CronJob/metallb-pool-drift-guard` now uses a dedicated service account with cluster-scoped `nodes` read permission, and the script’s `awk` expression for node subnet prefix extraction is fixed.
- **2026-01-07 – MetalLB smokes (IP assignment + drift guard):** Added `Job/metallb-smoke` (PostSync hook) and `CronJob/metallb-pool-drift-guard` under `platform/gitops/components/networking/metallb/tests/` to validate VIP assignment and detect pool↔node subnet drift.
- **2026-01-07 – memberlist encryption bootstrap:** Added `Job/metallb-memberlist-secret` (PostSync hook) under `platform/gitops/components/networking/metallb/tests/` to create `Secret/metallb-system/memberlist` when missing and restart MetalLB workloads to pick it up.
- **2025-12-29 – `metallb-controller` CrashLoopBackOff under access-guardrails self-protection:** MetalLB’s webhook cert rotator attempts to `UPDATE` `CustomResourceDefinition/*.metallb.io` (and `ValidatingWebhookConfiguration/metallb-webhook-configuration`) to inject/rotate the `caBundle`, but the self-protection guardrail denied those writes, causing the controller manager to exit and liveness probes to fail. Fix: allowlist `system:serviceaccount:metallb-system:metallb-controller` for those narrow `UPDATE`s.
