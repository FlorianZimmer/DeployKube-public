## Accepted / Won't Fix

### ConfigMap Replacement Strategy

- **Issue**: The component replaces the entire `coredns` ConfigMap rather than patching specific server blocks.
- **Status**: Accepted—the current config includes all standard blocks (health, ready, kubernetes, forward, cache, loop, reload, loadbalance). Upstream changes are rare and monitored during Kubernetes upgrades.

### Hardcoded PowerDNS IPs

- **Issue**: Forward targets are hardcoded IPs (`203.0.113.242`, `10.96.52.64`) rather than hostnames.
- **Status**: Intentional—avoids circular DNS dependency (CoreDNS cannot resolve a hostname to find the DNS server).

### Plain DNS Transport

- **Issue**: No TLS/DoT/DoH for DNS traffic.
- **Status**: Acceptable—internal cluster DNS is trusted; encryption would add complexity without meaningful security benefit.

## Resolved Issues

- _2026-03-16_ — **Phase 2 ownership handoff step 1:** Marked `ConfigMap/kube-system/coredns` with `argocd.argoproj.io/sync-options: Prune=false` so the later removal from the CoreDNS app cannot delete the live CoreDNS config during Argo prune.
- _2026-03-16_ — **Phase 1 CoreDNS upgrade-drift guardrail:** Added `platform/gitops/components/networking/coredns/upstream-corefile-baseline.yaml` plus `./tests/scripts/validate-coredns-upstream-corefile-contract.sh` so Kubernetes/Talos CoreDNS root-block changes become an explicit review step in CI. This documents the phased path while leaving the longer-term stub-block-only ownership refactor open.
- _2026-03-15_ — **Docs/runtime alignment and hook hardening:** Documented the tenant-provisioner-driven CoreDNS wiring path in the CoreDNS README and DNS design doc, corrected the artefact/runtime sections, removed the stale legacy open-placeholder block, and hardened `coredns-smoke` / `coredns-corefile-repair` with explicit non-root security contexts plus CPU/memory requests.
- _2026-03-15_ — **Hook image digest pinning:** Pinned `coredns-smoke` and `coredns-corefile-repair` to immutable digests, recorded the third-party `busybox` ref in the runtime artifact index, and extended the DNS centralized Trivy contract so the pinned refs stay visible in CI.
- _2026-03-15_ — **Hook scheduling tolerations:** Added standard `control-plane` / `master` `NoSchedule` tolerations to the repair and smoke Jobs so Argo hooks remain schedulable on tainted kube-system/control-plane nodes without forcing hard affinity.
- _2026-03-16_ — **Version-lock coverage promoted from gap to shared:** CoreDNS now reuses the shared `bootstrap-tools-image-tag` lock entry for its repo-owned hook-image baseline, so the component no longer carries the generic `versions.lock.yaml` gap marker. The third-party `busybox` smoke image remains curated through the runtime artifact index and Trivy contract rather than `versions.lock.yaml`.
- _2026-01-07_ — **Smoke Job implemented:** `PostSync Job/coredns-smoke` validates `cluster.local` resolution plus stub-domain and wildcard forwarding via CoreDNS (platform hostnames resolve consistently).
- _2026-01-29_ — **DNS wiring controller cutover:** Retired DeploymentConfig-rendered overlays and moved CoreDNS stub-domain forwarding to tenant provisioner reconciliation.
- _2026-01-31_ — **CoreDNS crashloop remediation:** CoreDNS can crashloop if duplicate stub-domain server blocks exist (e.g. from legacy Corefile content plus controller-managed stub block). Added a `PreSync` repair Job that removes duplicate stub-domain blocks outside the managed marker region before the smoke runs.
