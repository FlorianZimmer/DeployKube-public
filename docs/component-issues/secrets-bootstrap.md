### Component Assessment Findings (Automated)

#### High
#### design-doc-vs-implementation-drift
- Root-of-trust posture can silently default to low-assurance `inCluster` when DeploymentConfig is missing/unreadable (or tooling is unavailable), despite design proposing explicit prod opt-in (assurance + acknowledge fields); align by failing fast and/or enforcing the opt-in via schema/admission/CI validation (ids: `dk.ca.finding.v1:secrets-bootstrap:e724cf4b7500fe886c4aaca33a59ae173a4e281519b3ac8150c4ba52c7c60d95`).

#### non-root-containers
- Job lacks least-privilege hardening (non-root + seccomp + drop caps + no privilege escalation, etc.); add pod/container `securityContext` defaults (ids: `dk.ca.finding.v1:secrets-bootstrap:ea7b4e47ba9f6fb82a9b7c82952b90f99e379317f0a72de1f3d3d06ab38307f2`, `dk.ca.finding.v1:secrets-bootstrap:479dde65dfdbcaa699e0433972fb9d25d34e14d908888800ca9fcc81d3e03a8b`, `dk.ca.finding.v1:secrets-bootstrap:f935fd5e09f7ce8d0ce7a932d3da62e9d6bb9e3287cbaced76ae72d6b185f3df`).

#### rbac-implementation
- `secrets-bootstrap` identity can mutate Secrets cluster-wide; replace ClusterRole/ClusterRoleBinding with namespace-scoped Roles/RoleBindings only in `vault-system` and `vault-seal-system`, and restrict via `resourceNames` where feasible (ids: `dk.ca.finding.v1:secrets-bootstrap:5a6cd7ec6d52887b1f99cc7a8786786877694453f8465de3336fdf3633ca6a30`, `dk.ca.finding.v1:secrets-bootstrap:72b81fe7de5c55eec4156a56c634bcb1374ac1698ec194302ce881e5868de21b`, `dk.ca.finding.v1:secrets-bootstrap:69e7ef7fe4ea6627850e7a718b5a2b745dd50016419cf9dff9c4e6b50de2f772`, `dk.ca.finding.v1:secrets-bootstrap:8d1ef0922a1f059ba13e5a873ecd7d0ec52041da93ca2d71936240d430d3e8b5`, `dk.ca.finding.v1:secrets-bootstrap:2301cfeb29b4389c55e3d1e9a86d6963da25b61799297c6b8b8eba6cba81a743`, `dk.ca.finding.v1:secrets-bootstrap:f3f391809b74a1981cbb7138971ca9a46896d153eae73867852718aa2dfa67b2`).

#### Medium
#### backup-and-recovery
- Backup/restore guidance is high-level; add a minimal, testable restore procedure (preconditions, how to re-run the job) plus a post-restore verification checklist for required Secrets (ids: `dk.ca.finding.v1:secrets-bootstrap:bd6e28cddd052dc21e8b850456f3df4c63618022bf1b74db9a68424cc4064f13`).
