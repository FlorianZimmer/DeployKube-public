### Critical

#### high-availability
- Single-replica + `Recreate` rollout makes downtime unavoidable on drains/evictions/upgrades (cold start can be ~30min); document the expected downtime window + maintenance procedure, and consider a standby/failover model if interruptions must be avoided. (ids: `dk.ca.finding.v1:minecraft-monifactory:a654a57cf168000fb8bd4a81c6c4b93443e5a0b873a0d1c3fb4f0950a91de693`)

### High

#### design-doc-vs-implementation-drift
- Access management docs claim continuous Vault→RCON reconciliation, but implementation is day-0 seed + marker then in-game state; update the design doc and document the breakglass re-seed path (delete marker + restart). (ids: `dk.ca.finding.v1:minecraft-monifactory:950a1b1e8e96c04ef637e5877d7c85609e20fbfca6335a937650c5c47a191a83`)

#### general-security-pass
- Pod likely mounts the default ServiceAccount token; use a dedicated `ServiceAccount` and set `automountServiceAccountToken: false` when Kubernetes API access isn’t needed. - [x] **Baseline server parameters declared (2026-01-03):** `MAX_PLAYERS`, `VIEW_DISTANCE`, `SIMULATION_DISTANCE`, and a pinned `LEVEL_SEED` are GitOps-controlled via the generated `minecraft-monifactory-config` ConfigMap.
- [x] **Pack-authored config drift (2026-01-11):** Some mods (notably TelePastries + Create) persist default configs on the PVC that differ from the Monifactory server pack’s intended settings (e.g., Nether Cake requires hunger; Create worldgen not disabled). Fixed by adding the `monifactory-pack-config-sync` initContainer that extracts an allowlist of configs (plus `defaultconfigs/`) from the pinned server pack zip and copies them into `/data/config` and `/data/defaultconfigs`.
- [x] **Follow-up pack alignment (2026-01-11):** Converged remaining audited drifts (Iron Jetpacks tier configs, Lost Cities profile + custom disable, HammerLib tag mapping) and recorded a post-alignment audit summary.

Reduced the CPU request to `cpu=1` (kept `limits cpu=6`) to avoid starving other platform workloads. Memory remains high (`requests 10Gi`, `limits 12Gi`) because the pack can actually use ~11Gi working set.
